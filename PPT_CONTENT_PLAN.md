## 第2页：研究背景与问题提出

### 标题
**研究背景：僵尸网络威胁态势严峻**

### 内容要点

**1. 僵尸网络威胁现状**
- 僵尸网络(Botnet)是当前最具威胁的网络攻击基础设施
- 2025年全球僵尸网络感染设备超过**1500万台**
- 造成DDoS攻击、数据窃取、勒索软件传播等重大损失

**2. 传统检测方法的局限性**
| 方法类型 | 主要局限 |
|---------|---------|
| 基于签名 | 无法检测未知威胁 |
| 基于统计 | 忽视网络拓扑结构 |
| 传统ML | 特征工程依赖专家经验 |

**3. 核心挑战**
- **数据极度不平衡**：恶意流量占比<1%
- **隐蔽性强**：C&C通信伪装成正常流量
- **规模大**：单场景流量超千万条

## 第4页：核心方法 - 异构图构建与特征工程

### 标题
**核心方法一：异构图构建与深度特征提取**

### 内容要点

**1. 异构信息网络(HIN)设计**
- **节点类型**：IP节点、Protocol节点、Port节点
- **边类型**：IP-IP(Flow)、IP-Protocol、IP-Port
- **边权重策略**：通信频次对数映射 `log1p(count)`

**2. 64维深度特征矩阵**
| 特征维度 | 特征类别 | 具体内容 |
|---------|---------|---------|
| 0-4 | 网络拓扑 | 唯一通信对端数、端口离散度 |
| 5-17 | 包/字节统计 | 均值、方差、极值、变异系数 |
| 18-28 | 速率特征 | PPS、BPS、BPP |
| 29-33 | 时序特征 | IAT均值、标准差 |
| 34-36 | 协议分布 | TCP/UDP/ICMP占比 |
| 37-43 | 端口语义 | 知名端口/高位端口使用率 |

**3. 特征预处理**
- Log1p变换处理长尾分布
- Z-Score标准化消除量纲差异

## 第5页：核心方法 - GNN检测引擎

### 标题
**核心方法二：GAT-GCN混合图神经网络**

### 内容要点

**1. 模型架构设计**

```
Input Features (64-D)
        ↓
┌───────────────────────────┐
│   GAT Layer (4 Heads)     │  ← 多头注意力机制
│   学习节点间语义关联        │
└───────────────────────────┘
        ↓
┌───────────────────────────┐
│   GCN Layer               │  ← 图卷积聚合
│   聚合局部邻域信息          │
└───────────────────────────┘
        ↓
┌───────────────────────────┐
│   BatchNorm + PReLU       │  ← 正则化
│   + Dropout(0.4)          │
└───────────────────────────┘
        ↓
    Output (Bot Score 0-1)
```

**2. 关键技术创新**

| 技术 | 解决问题 | 效果 |
|-----|---------|-----|
| **Focal Loss** (α=0.8, γ=2.0) | 样本极度不平衡 | 聚焦难分类样本 |
| **NeighborLoader** | 大规模图训练 | 显存优化80%+ |
| **混合精度训练(AMP)** | 训练速度 | 加速2-3倍 |
| **多头注意力(Heads=4)** | 特征表达 | 捕获多角度关系 |

**3. 与基线方法对比**
- Bot-AHGCN: 需计算O(n²)相似度矩阵，内存消耗大
- 本方法: 采样策略线性复杂度，支持增量更新

- 隐马尔可夫模型识别攻击阶段

## 第9页：实验结果与分析

### 标题
**实验结果：GAT-GCN在CTU-13上的检测性能**

### 内容要点

**1. 核心性能指标（真实运行结果）**

| 方法 | AUC | F1-Score | Precision | Recall |
|-----|-----|----------|-----------|--------|
| **GAT-GCN (Ours)** | **0.846** | 0.019 | 0.010 | **1.000** |
| Bot-AHGCN (Baseline) | 0.824 | 0.051 | 0.026 | 0.978 |

**注**: 以上数据均为真实运行结果，训练场景1-5，测试场景13。

**2. 关键发现**

✅ **AUC提升2.7%**：GAT-GCN (0.846) vs Bot-AHGCN (0.824)

✅ **召回率达到100%**：所有僵尸节点均被成功识别

✅ **跨场景泛化能力**：在未见过的Scenario 13上保持高性能

⚠️ **挑战**：精确率较低反映数据极度不平衡（Bot占比<1%）

**3. 结果分析**
- 极端数据不平衡是核心挑战：恶意流量仅占约1%
- 两种GNN方法在召回率上都表现优异（>97%）
- GAT-GCN通过注意力机制提升了AUC，但精确率仍有改进空间
- 后续可通过改进阈值策略或代价敏感学习优化精确率
